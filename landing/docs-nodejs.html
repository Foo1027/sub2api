Node.js 环境安装教程

Install

在 Windows、macOS、Linux 安装 Node.js LTS，并验证 node/npm 命令可用。

💡 提示：后续的 Claude Code、Gemini CLI、Codex 都依赖 Node.js 运行环境。先完成本教程，再继续各自工具配置。

Windows

方法一：官方下载

前往 Node.js 官网 下载 LTS 版本，双击安装包按提示安装即可。

方法二：使用 Chocolatey

CMD / PowerShell
choco install nodejs-lts

方法三：使用 Scoop

CMD / PowerShell
scoop install nodejs-lts

验证安装

TERMINAL
node --version
npm --version

Linux

Ubuntu / Debian

BASH
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs

CentOS / RHEL

BASH
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs

验证安装

TERMINAL
node --version
npm --version

如果能输出版本号，说明 Node.js 与 npm 已可用。

macOS

官方文档里 macOS 明确给了两种方式：Homebrew 和官网下载。前者更省事，后者适合不使用 Homebrew 的机器。

方法一：使用 Homebrew

BASH
brew install node
node --version
npm --version

方法二：官方下载

前往 Node.js 官网 下载 macOS 安装包。

其他方式

如果你需要在不同项目之间切换 Node 版本，推荐使用 nvm 进行版本管理。

BASH
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.nvm/nvm.sh
nvm install --lts
nvm use --lts
node --version
npm --version

Gemini CLI 配置教程

Install

按系统安装 Gemini CLI

开始前请先完成 Node.js 环境安装。

CMD / PowerShell
npm install -g @google/gemini-cli
gemini --version
BASH
sudo npm install -g @google/gemini-cli
gemini --version
BASH
sudo npm install -g @google/gemini-cli
gemini --version
HOMEBREW
brew install gemini-cli
gemini --version
Environment

配置环境变量

设置环境变量让 Gemini CLI 连接到 APIKEY.FUN 平台。需要设置三个变量：
GOOGLE_GEMINI_BASE_URL -- 服务地址
GEMINI_API_KEY -- 你的 API Key
GEMINI_MODEL -- 使用的模型

Temporary

临时设置只在当前终端会话生效，适合先快速验证。

POWERSHELL
$env:GOOGLE_GEMINI_BASE_URL = "https://aifoo.cc.cd"
$env:GEMINI_API_KEY = "YOUR_API_KEY"
$env:GEMINI_MODEL = "gemini-3-pro-preview"
BASH
export GOOGLE_GEMINI_BASE_URL="https://aifoo.cc.cd"
export GEMINI_API_KEY="YOUR_API_KEY"
export GEMINI_MODEL="gemini-3-pro-preview"
ZSH / BASH
export GOOGLE_GEMINI_BASE_URL="https://aifoo.cc.cd"
export GEMINI_API_KEY="YOUR_API_KEY"
export GEMINI_MODEL="gemini-3-pro-preview"
Permanent

永久设置适合长期使用，重启终端后也能自动生效。

POWERSHELL
[System.Environment]::SetEnvironmentVariable("GOOGLE_GEMINI_BASE_URL", "https://aifoo.cc.cd", [System.EnvironmentVariableTarget]::User)
[System.Environment]::SetEnvironmentVariable("GEMINI_API_KEY", "YOUR_API_KEY", [System.EnvironmentVariableTarget]::User)
[System.Environment]::SetEnvironmentVariable("GEMINI_MODEL", "gemini-3-pro-preview", [System.EnvironmentVariableTarget]::User)
BASH
echo 'export GOOGLE_GEMINI_BASE_URL="https://aifoo.cc.cd"' >> ~/.bashrc
echo 'export GEMINI_API_KEY="YOUR_API_KEY"' >> ~/.bashrc
echo 'export GEMINI_MODEL="gemini-3-pro-preview"' >> ~/.bashrc
source ~/.bashrc
ZSH
echo 'export GOOGLE_GEMINI_BASE_URL="https://aifoo.cc.cd"' >> ~/.zshrc
echo 'export GEMINI_API_KEY="YOUR_API_KEY"' >> ~/.zshrc
echo 'export GEMINI_MODEL="gemini-3-pro-preview"' >> ~/.zshrc
source ~/.zshrc

💡 提示：请将命令中的 YOUR_API_KEY 替换为控制台里复制的任意 API Key。如需更换模型，只需修改 GEMINI_MODEL 即可。

Run

开始使用

配置完成后，在项目目录中运行以下命令即可开始使用 Gemini CLI。

TERMINAL
gemini

如果想先做一次非交互测试，可以直接运行下面这条命令。

TERMINAL
gemini -p "你好"

OpenClaw 配置教程

推荐使用一键脚本，按提示选择 Anthropic（Claude）或 OpenAI（Codex）通道。

Anthropic（Claude）

https://aifoo.cc.cd

Claude / Anthropic 接口不要带 /v1，示例模型可用 claude-opus-4-6。

OpenAI（Codex）

https://aifoo.cc.cd/v1

OpenAI / Codex 接口必须带 /v1，示例模型可用 gpt-5.5。

Quick Start

推荐方式：交互式一键脚本

运行后选择通道并输入 API Key，脚本会写入 OpenClaw 配置。

BASH
curl -fsSL https://aifoo.cc.cd/install/openclaw.sh | sudo bash

💡 提示：脚本执行后会提示输入 API Key。请先在控制台获取对应分组的密钥，再按提示粘贴。

备用方案：手动安装与配置 OpenClaw一键脚本不可用,想先审查命令或希望自己控制每一步时再展开

第一步：安装 OpenClaw

BASH
npm install -g @openclaw/cli

第二步：选择接入通道

BASH
export ANTHROPIC_API_KEY="YOUR_API_KEY"
openclaw onboard --auth-choice custom-api-key \
  --custom-base-url https://aifoo.cc.cd \
  --custom-api-key-env ANTHROPIC_API_KEY \
  --custom-compatibility anthropic \
  --custom-model claude-opus-4-6

Anthropic（Claude）通道使用根地址，不要加 /v1。

BASH
export OPENAI_API_KEY="YOUR_API_KEY"
openclaw onboard --auth-choice custom-api-key \
  --custom-base-url https://aifoo.cc.cd/v1 \
  --custom-api-key-env OPENAI_API_KEY \
  --custom-compatibility openai \
  --custom-model gpt-5.5

OpenAI（Codex）通道使用带 /v1 的地址。

💡 提示：请先在控制台获取对应分组的 API Key，并替换命令中的 YOUR_API_KEY。

第三步：开始使用

TERMINAL
openclaw

Hermes 配置教程

Quick Start

推荐方式：交互式一键脚本

运行后选择通道并输入 API Key，脚本会写入 ~/.hermes/config.yaml。

BASH
curl -fsSL https://aifoo.cc.cd/install/hermes.sh | sudo bash

💡 提示：脚本执行后会提示输入 API Key。请先在控制台获取对应分组的密钥，再按提示粘贴。

备用方案：手动安装与配置 Hermes一键脚本不可用,想先审查命令或希望自己控制每一步时再展开

注意：Claude / Anthropic 接口不要带 /v1；OpenAI / Codex 接口必须带 /v1。

第一步：安装 Hermes

BASH
pipx install hermes-agent

第二步：选择接入通道

YAML
mkdir -p ~/.hermes
cat > ~/.hermes/config.yaml << 'EOF'
model:
  default: claude-opus-4-7
  provider: aifoo-claude
providers:
  aifoo-claude:
    api_mode: anthropic_messages
    base_url: https://aifoo.cc.cd
    api_key: YOUR_API_KEY
    default_model: claude-opus-4-7
    models:
      - claude-opus-4-7
EOF
YAML
mkdir -p ~/.hermes
cat > ~/.hermes/config.yaml << 'EOF'
model:
  default: gpt-5.5
  provider: aifoo-codex
providers:
  aifoo-codex:
    api_mode: codex_responses
    base_url: https://aifoo.cc.cd/v1
    api_key: YOUR_API_KEY
    default_model: gpt-5.5
    models:
      - gpt-5.5
EOF

💡 提示：请先在控制台获取对应分组的 API Key，并替换命令中的 YOUR_API_KEY。

第三步：开始使用

TERMINAL
hermes

API 脚本接入教程

下面示例覆盖文本、视觉和图片接口。复制后把 YOUR_API_KEY 替换为你的密钥即可运行。

Overview
OpenAI Responses

适合新接入或直接对齐新一代 OpenAI 风格工作流。

Chat Completions

适合已有旧版 OpenAI 客户端和历史脚本的迁移。

Anthropic Messages

适合明确走 Claude 通道的消息式调用。

图片接口

额外补了图片生成和图片编辑脚本，方便直接接视觉能力。

如果你是全新接入，建议优先看 OpenAI Responses；如果你已经有旧版 OpenAI 客户端，优先看 Chat Completions；如果你接的是 Claude 通道，就看 Anthropic Messages。
